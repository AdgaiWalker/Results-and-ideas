# Walker 个人博客设计规格

## 项目概述

为 Walker 打造一个前卫实验风格的个人博客，涵盖技术、旅行、文艺、生活、设计、厨艺、写作、哲学等多领域内容。以「行船」为核心意象，传达持续探索、不设边界的精神。

---

## 品牌与视觉

### 品牌名
**Walker**

### 核心意象
行船——孤帆在夜海中航行，金色光芒破云而出。作为首页视觉焦点（点睛之笔），其他页面以「水」的感知贯穿。

### 色彩体系
| 角色 | 色值 | 用途 |
|------|------|------|
| 底色 | 深靛蓝/藏青 `#0a1628` | 背景、大面积色块 |
| 强调色 | 金色/琥珀 `#d4af37` | 高亮、链接、哲学标签 |
| 辅助色 | 冷灰 `#4a5568` | 分割线、次要文字、边框 |
| 主文字 | 暖白 `#f0ead6` | 正文内容 |
| 次要文字 | 淡灰 `#9ca3af` | 日期、标签、说明文字 |

### 字体
- 标题：**Sora**（几何感、现代、有辨识度但不张扬）
- 正文：**Noto Sans SC**（中文优先，高可读性）
- 代码：**JetBrains Mono**

## 站点架构

### 页面结构（三个核心空间）

#### 1. 港口（首页 `/`）

**首屏：**
- 全屏视频背景（AI 生成的行船动画，自动播放、静音、无限循环）
- 视频来源：基于用户提供的「夜海孤帆」静态图，通过 AI 视频工具生成
- 叠加内容：「Walker」+ 一句短标语，文字带投影保证可读性
- 底部渐变条（透明→深靛蓝），与页面背景自然融合
- 弱网/移动端流量降级为原始静态图
- 用户向下滚动时视频逐渐淡出

**首屏以下：**
- 「最近痕迹」区域：最近 6-8 条内容，以卡片形式交错排列
  - 不同 type 的卡片有不同视觉形态（文章→紧凑卡、图集→宽幅图卡、随想→文字条目）
  - 无分类隔离，按时间排列，混合呈现
- 底部个人区：一段关于 Walker 的简短文字，像港口立的一块小牌子

#### 2. 航海日志（内容流 `/log/`）

博客主体。所有已发布内容按时间倒序排列。

**设计要点：**
- 无固定分类，全部通过标签（tags）自由标记
- 支持按标签筛选（顶部水平筛选条，横向滚动，移动端友好）
- 不同 type 呈现不同形态：
  - `article`：带代码高亮的长文，标准博客文章排版
  - `photo`：全宽图集 + 短文描述
  - `thought`：一句话或一小段，日记条目风格，视觉更轻
  - `project`：项目展示卡，带外部链接和截图
- 分页加载（每页 12 条，「加载更多」按钮，避免无限滚动的性能问题）

#### 3. 指南针（探索 `/compass/`）

替代传统 About 页面的存在。

**设计要点：**
- 标签图谱：所有使用过的标签以可视化方式呈现（非传统标签云）
  - 标签大小反映使用频次
  - 哲学标签用金色（强调色）高亮，暗示贯穿一切的线索
  - 点击标签展开该标签下的所有内容
- 兴趣统计：显示各标签下的内容数量，通过数据定义「你是谁」
- 可选：一条简短的个人宣言

## 内容管理

### 文件结构

```
Obsidian 仓库/
  ├── ...其他笔记...
  ├── walker/                    ← 博客内容源（Obsidian 可直接编辑）
  │   ├── 夜海上的念头.md
  │   ├── React 性能优化笔记.md
  │   └── ...
```

Astro 的内容配置通过路径映射指向 Obsidian 仓库中的 `walker/` 文件夹。实现方式：在 `astro.config.mjs` 中使用 symlink 或直接配置 content collections 的路径指向 Obsidian vault 中的对应目录。构建时 Astro 直接读取 Obsidian 中的 Markdown 文件。

### Markdown Frontmatter 规范

```yaml
---
title: 标题
date: 2026-04-29
tags: [哲学, 随想]     # 自由标签，无预定义列表，随写随加
type: thought           # article | photo | thought | project
published: true         # true 才会出现在博客上，false 为草稿
cover: /images/xxx.jpg  # 可选，封面图
---
```

### 发布流程

1. 在 Obsidian 中写作，填写 frontmatter
2. `git add . && git commit -m "新日志" && git push`
3. 自动构建部署，发布完成

### 标签系统
- 标签不预定义，写作时自由添加
- 新标签自动出现在「指南针」页面的图谱中
- 一篇内容可有多个标签（支持跨领域，如 `[技术, 哲学]`）
- 标签列表随时间自然生长，反映真实兴趣轨迹

## 交互设计

### 全局
- 页面切换：水波扩散式过渡动画（克制，不喧宾夺主）
- 滚动动效：内容卡片进入视口时淡入上移
- 悬停反馈：卡片 hover 时轻微浮起 + 阴影加深
- 响应式设计：适配桌面端和移动端

### 无障碍
- 所有动画支持 `prefers-reduced-motion` 降级
- 视频降级方案（弱网→静态图）
- 足够的文字对比度
- 键盘可导航

## 技术方案

### 框架
**Astro** — 静态站点生成器
- 理由：Markdown 原生支持、静态优先性能极佳、Islands 架构精确控制交互区域、构建速度快

### 样式
**Tailwind CSS** — 原子化 CSS 框架
- 理由：与 Astro 配合良好、快速实现自定义设计、减少自定义 CSS 体积

### 动画
- 页面过渡：CSS transitions + 少量 View Transitions API
- 滚动动效：Intersection Observer（CSS + 轻量 JS）
- 首页视频：原生 `<video>` 标签，autoplay + muted + loop

### 部署
- **Vercel**：推送即部署，零配置，Astro 官方推荐
- 自定义域名可选
