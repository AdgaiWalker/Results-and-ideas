# Walker 知识空间重构设计文档

> **目标：** 将 Walker 从传统博客架构重构为 AI 时代个人知识空间——以概念图谱为核心、支持框架进化、同时服务于自己、人类访客和 AI agent。

---

## 一、核心目的

| # | 服务对象 | 目的 | 场景 |
|---|---------|------|------|
| 1 | 自己 | 知识存储和检索 | Obsidian 写 → 自动同步到站 |
| 2 | 人类访客 | 建立信任，传递知识和 idea | 来看 → 感受到广度深度人格 → 带走有用的东西 |
| 3 | AI agent | 作为知识渠道服务他人 | 别人给 AI 你的网址 → AI 检索你的知识帮别人解决问题 |

**统一体验目标：** 走的时候觉得「这个人有广度、有深度、有意思、想得清楚」

---

## 二、核心哲学 — 五层逻辑（连续循环系统）

五层不是五个独立原则，是一个有方向的循环系统：

```
① 人类主权 ──→ 人决策，AI 执行。人可以授权，AI 在授权范围内自由行动
      ↓
② 熵减命令 ──→ 目标：高熵（混乱）→ 低熵（清晰）
      ↓
③ 多元世界观 ──→ 方法：人写蓝图（应然），AI 转化为现实（实然），偏差驱动迭代
      ↓
④ 量变质变 ──→ 节奏：写蓝图→执行→对比→调整，循环积累到突破
      ↓
⑤ 做减法 ──→ 验证：最终结果是否让人的工作更少了？
      ↓
   是 → 完成
   否 → 回溯到对应层，循环
```

**第③层是核心操作层：** 在 AI 作为劳动力的前提下，人写方案/文档/意图 → AI 转化为代码/产出/现实服务。随着 AI 智能加深，这个循环的力度越来越大。

**第⑤层是检验关卡：** 前四层走下来，如果人的负担没减轻，说明某个环节有问题。

## 三、架构核心转变

```
从：博客结构（时间线 + 分类标签 + 固定页面）
到：知识图谱（概念节点 + 内容 + 链接关系 + 可进化框架）
```

### 四层模型

```
┌─────────────────────────────────────────────┐
│  体验层                                      │
│  行船意象 · 审美是长出来的 · 前卫视觉          │
│  View Transitions · scroll-driven animations │
├─────────────────────────────────────────────┤
│  视图层                                      │
│  Obsidian（编辑）· 网站（浏览）· 结构化数据（AI）│
├─────────────────────────────────────────────┤
│  数据层                                      │
│  概念节点 ←→ 内容（任何媒体类型）←→ 概念节点    │
│  框架 = 当前活跃的顶层概念集合                  │
├─────────────────────────────────────────────┤
│  性能层                                      │
│  Pretext 排版 · 虚拟滚动 · Astro 静态生成      │
└─────────────────────────────────────────────┘
```

### 经纬结构

```
框架（深度·经线）— 五层逻辑等顶层概念，是连续循环系统
  │
  ├─ 连接到厨艺内容
  ├─ 连接到技术内容
  ├─ 连接到设计内容
  └─ 连接到生活内容

领域（广度·纬线）— 厨艺、技术、音乐、摄影……
  │
  ├─ 入口：访客因为某个领域点进来
  └─ 深度：点进来后发现背后有统一的思维体系
```

### 领域自然扩张

明天迷上陶艺 → Obsidian 新建 `ceramics.md` → 站上自动出现新节点。
不需要建板块、改导航、加页面。**加一个概念文件就是在扩展世界。**

## 四、数据模型

### 4.1 内容源：Obsidian

Obsidian vault 作为唯一数据源。站是 Obsidian 的公开视图。

**工作流：**
```
Obsidian 编辑 → Git push → Astro 构建 → Vercel 部署
```

**Obsidian vault 结构：**
```
walker-vault/
├── 00-概念/        # 概念节点（对应 concepts/）
├── 01-文章/        # 长文
├── 02-随想/        # 短文本
├── 03-对话/        # AI 对话记录
├── 04-食谱/        # 厨艺
├── 05-提示词/      # 可分享的 prompt
├── 06-项目/        # 项目记录
├── 07-视频/        # 视频内容元数据
├── 08-音频/        # 音频内容元数据
├── 09-图片/        # 图片集元数据
└── 99-草稿/        # 未发布的内容（published: false）
```

**站内容目录（Astro 读取）：**
```
src/content/
├── concepts/           # 概念节点
├── articles/           # 长文
├── thoughts/           # 随想
├── dialogues/          # AI 对话
├── recipes/            # 食谱
├── prompts/            # 提示词
├── projects/           # 项目
├── videos/             # 视频元数据
├── audios/             # 音频元数据
└── galleries/          # 图片集元数据
```

### 4.2 概念节点（Concept）

概念是站的核心单元。每个概念是一个 Markdown 文件：

```markdown
---
title: 做减法
type: concept
symbol: −
domain: [哲学]
related: [human-sovereignty, entropy-reduction]
layer: 5
status: active
---

如无必要，勿增实体。最简路径即最优路径。

这条原则体现在我生活的方方面面——从代码架构到食材选择到提示词精炼。
```

**关键属性：**
- `related` — 双向链接，表达概念间关系
- `domain` — 所属领域（厨艺、技术、哲学…）
- `status` — active 的概念显示在框架层，archived 的保留但不突出
- `symbol` — 框架概念的可视符号
- `layer` — 在五层逻辑中的层级（1-5，仅框架概念有）

### 4.3 内容类型与多媒体

所有媒体类型通过 `concepts` 统一挂到概念节点上。图谱不关心媒体类型，只关心知识的连接。

| 类型 | 存放目录 | 视觉表达 | 特点 |
|------|---------|---------|------|
| **article** | articles/ | 深度长文排版 | 有标题、摘要、正文 |
| **thought** | thoughts/ | 卡片式，列表直接读完 | 一句话到三句话，不需要点进去 |
| **dialogue** | dialogues/ | 人机气泡对话 | 附带收获提炼，可折叠长对话 |
| **recipe** | recipes/ | 食材表 + 步骤 + 思考 | 不是叙事散文 |
| **prompt** | prompts/ | 独立可复制块 | 自带场景、原则、效果说明 |
| **project** | projects/ | 成果展示 + 关键决策 | 不是文章 |
| **video** | videos/ | 嵌入播放器 + 思考 + 关联概念 | 视频文件放 public/ |
| **audio** | audios/ | 播放器 + 为什么分享 + 关联概念 | 音频文件放 public/ |
| **gallery** | galleries/ | 瀑布流画廊 + 背景 + 关联概念 | 图片放 public/ |

### 4.4 内容与概念的关联

内容通过 frontmatter 的 `concepts` 字段关联到概念节点：

```markdown
---
title: AI 辅助设计的工作流
date: 2026-04-26
type: article
concepts: [subtraction, human-sovereignty, entropy-reduction]
tags: [技术, 设计, AI]
---
```

`concepts` 是结构化的概念 ID，`tags` 是自由标签。概念页面自动聚合关联它的所有内容。

### 4.5 对话工作流（人类主权 = 最小工作量）

**DeepSeek 对话：** 粘贴分享链接到 frontmatter 的 `source` 字段 = 授权。构建时自动抓取解析渲染。

```markdown
---
title: 和 DeepSeek 聊做减法
date: 2026-04-25
concepts: [subtraction]
source: https://chat.deepseek.com/share/xxx
---

## 收获 ← 可选
做减法不是少做，是想清楚再做。
```

**Obsidian AI 对话：** 直接在 Obsidian 中聊完，构建时自动解析角色格式。

**可选标记：** 加 `★` 标记重点轮次，其余折叠。不加就全部展开。

**导出工具：** [DeepSeek-Chat-Exporter](https://github.com/blueberrycongee/DeepSeek-Chat-Exporter) 油猴脚本，一键导出 Markdown。

### 4.6 自动化（你只做决策）

| 你做的（决策） | 系统自动做的（执行） |
|--------------|-------------------|
| 写内容，标注 `concepts: [subtraction]` | 自动关联概念页面、更新图谱 |
| 粘贴 DeepSeek 分享链接 | 自动抓取、解析角色、渲染气泡 |
| `published: true` | 构建部署上线 |
| 新建一个概念文件 | 自动出现在图谱、框架、导航 |
| 拍一段视频放到 public/，写元数据 | 自动生成视频条目页面 |

## 五、页面结构

### 5.1 首页 — 世界观

不再是「最新 6 篇」。是 Walker 知识世界的全景俯瞰。

**结构：**
1. Hero 视频（保留，行船开场）
2. 知识图谱可视化 — 交互式概念关系图（D3.js force-directed）
   - 节点大小 = 关联内容数量
   - 节点颜色 = 领域
   - 线 = 概念间关系
   - 点击节点 → 进入概念页面
   - 悬停 → 预览关联内容
3. 五层逻辑流程线 — 作为可见的递进关系（①→⑤有方向箭头），最后一站标注「验证：人的工作更少了吗？」
4. 最近动态 — 3-4 条，安静地待在角落

**图谱不是静态的装饰，是站的核心导航方式。**

### 5.2 概念页面 — 核心导航单元

每个概念一个页面，是站的主要导航方式。

**结构：**
```
概念名称 + 定义
├── 关联概念（链接 + 关系说明）
├── 蓝图→现实循环（如果有）
│   ├── 意图（蓝图/应然）
│   ├── 产出（现实/实然）
│   ├── 偏差和调整
│   └── 结果验证
├── 关联内容（按媒体类型分组显示）
│   ├── 📄 文章
│   ├── 💬 对话
│   ├── 🍝 食谱
│   ├── 🎥 视频
│   ├── 🎵 音频
│   ├── 📷 图片集
│   └── 🔧 提示词（可复制）
└── 该概念下的随想（卡片式）
```

### 5.3 内容页面 — 各类型独立模板

- **article** — 深度排版，底部显示关联概念
- **thought** — 可能不需要独立页面，列表直接读完
- **dialogue** — 气泡对话 + 收获提炼 + 关联概念。长对话默认折叠，★标记的展开
- **recipe** — 食材表 + 步骤 + 思考（链接到概念）
- **prompt** — 独立页，自带场景说明和复制按钮
- **project** — 成果展示 + 关键决策
- **video** — 嵌入播放器 + 文字思考 + 关联概念
- **audio** — 播放器 + 分享理由 + 关联概念
- **gallery** — 瀑布流画廊 + 拍摄背景 + 关联概念

### 5.4 日志总览 — 按时间 + 按概念双入口

保留时间维度的瀑布流（上旬/中旬/下旬），同时增加概念维度的导航。

### 5.5 提示词库 — 独立页面

按概念和场景分类的提示词集合。每条提示词：
- 标题 + 内容（可复制）
- 所属概念
- 适用场景
- 效果说明

### 5.6 框架页面 — 五层逻辑的可视化

替代现在的静态指南针。展示五层逻辑的：
- 连续递进关系（流程图/时间线形式，非卡片列表）
- 每层在实际内容中的体现（链接到概念页面）
- 循环回溯的可视化
- 最后一层的验证问题：「你的工作更少了吗？」

## 七、设计原则（按优先级）

1. **审美是长出来的** — 视觉语言反映真实生命体验（走过的路、看过的人与景、经历的事与物、做的多与少），不是装饰。每个元素都有存在的理由
2. **人类主权** — 人决策授权，AI 在授权范围内自由执行。放链接 = 授权
3. **做减法** — 验证标准：结果是否让人的工作更少了
4. **框架可进化** — 概念增删改不需要重构站
5. **领域自然扩张** — 加概念文件就是扩展世界
6. **三种读者统一** — 一个数据结构，三种视图
7. **最小工作量** — 你只做决策，系统自动执行
8. **多媒体承载** — 所有媒体类型通过概念统一关联
9. **渐进增强** — 先保证内容和结构，交互和动画是锦上添花
10. **Pretext 就绪** — 从一开始就为大规模内容做准备
